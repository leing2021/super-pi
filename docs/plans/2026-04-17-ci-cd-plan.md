# Plan

## Problem summary

super-pi needs automated CI/CD. Requirements are in `docs/brainstorms/2026-04-17-ci-cd-requirements.md`. The chosen approach is two separate GitHub Actions workflows: `test.yml` for continuous testing and `publish.yml` for tag-triggered npm publish.

## Relevant learnings

None yet. `docs/solutions/` is empty.

## Scope boundaries

**In scope:**
- `test.yml` workflow: runs `bun test` on push/PR to main
- `publish.yml` workflow: runs `bun test` then `npm publish` on tag push `v*`
- README documentation of CI/CD
- `NPM_TOKEN` as GitHub secret

**Out of scope:**
- Automated version bumping
- Changelog generation
- Cross-platform matrix
- npm provenance / signing
- Release automation beyond tag-triggered publish

## Implementation units

### Unit 1: test.yml workflow

**Goal:** Create `.github/workflows/test.yml` that runs `bun test` on every push and PR to main.

**Files:**
- Create: `.github/workflows/test.yml`

**Patterns to follow:**
- Use `oven-sh/setup-bun@v2` for Bun installation
- Use `actions/checkout@v4`
- Pin Bun version to match local (`1.3.12`)
- Run `bun install` then `bun test`

**Test scenarios:**
- Push to main triggers the workflow
- PR to main triggers the workflow
- Workflow fails if tests fail
- Workflow passes if tests pass

**Verification:**
```bash
# Validate YAML syntax
cat .github/workflows/test.yml
# Push to GitHub and check Actions tab
```

**Dependencies:** None

---

### Unit 2: publish.yml workflow

**Goal:** Create `.github/workflows/publish.yml` that runs tests and publishes to npm when a `v*` tag is pushed.

**Files:**
- Create: `.github/workflows/publish.yml`

**Patterns to follow:**
- Trigger on tag push matching `v*`
- Reuse same Bun setup as test.yml
- Run `bun install` then `bun test` before publish
- Use `NPM_TOKEN` secret for authentication
- Use `npm publish --access public` (explicit for safety)

**Test scenarios:**
- Pushing `v0.1.2` tag triggers the workflow
- Workflow runs tests first
- If tests fail, publish does not execute
- If tests pass, `npm publish` runs with `NPM_TOKEN`
- Workflow skips on non-tag pushes

**Verification:**
```bash
# Validate YAML syntax
cat .github/workflows/publish.yml
# Push a test tag and check Actions tab
git tag v0.1.2
git push origin v0.1.2
# Then clean up if needed
```

**Dependencies:** Unit 1 (shares Bun setup pattern)
