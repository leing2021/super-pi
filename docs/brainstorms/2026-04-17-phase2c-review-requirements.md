# Requirements

## Problem

`ce-review` uses a flat reviewer selection process — the agent manually decides which reviewer personas to apply. There is no structured diff analysis to automatically route to the right reviewers. There is also no autofix capability — all findings must be manually addressed.

## Goals

- A `review_router` extension tool that analyzes diff metadata (file types, change size, affected paths) and returns a recommended list of reviewer personas
- Updated `reviewer-selection.md` with richer persona definitions and trigger conditions
- Updated `ce-review` SKILL.md with autofix guidance: when findings have clear fixes, the agent should apply them and re-review
- Updated `findings-schema.md` with autofix tracking fields
- Updated `handoff.md` to support autofix loop

## Non-goals

- No actual code patch generation by the tool (autofix is guided by SKILL.md, not generated by a tool)
- No parallel subagent execution for reviewers (still serial in Phase 2c)
- No CI/CD integration for review
- No review persistence beyond the current session

## Approach options

### A: Pure SKILL.md enhancement

Only update SKILL.md, findings-schema, reviewer-selection, handoff. All routing and autofix driven by agent interpretation.

Pros: simplest.
Cons: no reusable tool, no testable routing logic.

### B: review_router tool + SKILL.md autofix (recommended)

Add `review_router` tool for structured diff analysis and persona routing. Autofix guidance stays in SKILL.md.

Pros: reusable tool, testable routing, autofix stays flexible.
Cons: more code than A.

### C: review_router tool + autofix tool

Two new tools: one for routing, one for generating fix patches.

Pros: most complete.
Cons: autofix tool is hard to generalize, overkill for Phase 2c.

## Recommended direction

**Approach B**: `review_router` tool for structured diff-based reviewer routing + SKILL.md autofix guidance. The tool makes reviewer selection testable and reusable. Autofix is guided by instructions rather than code.

## Success criteria

- [ ] `review_router` tool analyzes diff metadata and returns recommended reviewer personas
- [ ] `review_router` considers file types, change size, and affected paths
- [ ] `ce-review` SKILL.md uses `review_router` for persona selection
- [ ] `ce-review` SKILL.md includes autofix loop guidance
- [ ] `findings-schema.md` updated with autofix tracking fields
- [ ] `reviewer-selection.md` updated with richer persona definitions
- [ ] `handoff.md` updated to support autofix loop
- [ ] All existing tests pass
- [ ] New tool has TDD tests
